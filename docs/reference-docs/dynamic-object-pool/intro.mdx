# 对象池类介绍

该对象池是基于 ``UnityEngine.Pool`` 进行扩展的，它支持多种不同类型的物体的对象池，并可以根据调用方法提供的不同参数进行分类。

## 什么是对象池？

对象池 (Object Pool)，是一种用于资源管理调度的设计模式[^1]。他可以轻松管理那些需要高频创建和删除的类型。它是一种**用内存换时间**的概念。  
高频创建一个物体时，通常会带来庞大的性能开销，到后面会发现，内存消耗越来越大，同时运行程序也越来越卡。这就是因为频繁创建和销毁物体导致的性能开销和内存释放问题。如果你也有这个问题，*用上对象池就完事了*！

## 为什么在Unity中要使用对象池？

在传统的C#或者Java开发中，最高频使用对象池的地方是HTTTP请求或者大开销多线程（或者高并发的情况），这时需要能高效管理所有请求或者线程，于是便有了专用于请求类型的对象池————线程池(Threading Pool)。  
Unity中的物体已经不仅仅是简单的一个类（当然物体上当然也可以称为一个类，不过是一个基于 `UnityEngine: Object` 的类），而是一系列脚本和组件（Component）的集合。因此更加需要合理的管理与分配。因此，对象池是游戏开发中经常使用到的技术[^2]。

## 何时需要在Unity中使用对象池？

对象池既是优点也是缺点。因为是***用内存换时间***，因此每开一个对象池都会伴随着巨大的内存消耗，因此使用时仅用于需要高频创建的物体，例如：
- 临时的GameObject，如子弹、弹孔平面贴图等
- 特效，如子弹飞行的拖尾特效
- UI，如加载了数据后的排行榜Prefab

使用对象池的物体有以下特点：
- 外观基本一致（不会说子弹一开始是金色尖头，后面就变成橙色方块）
- 多次在游戏中出现且频繁创建和销毁
- 与外部场景关联性弱，不强制依赖于某个仅在特定场景下的脚本或物体
